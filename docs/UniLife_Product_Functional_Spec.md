# **UniLife 产品功能详述文档 (Product Functional Specification)**

| 项目名称 | UniLife |
| :---- | :---- |
| **文档类型** | 功能详述 / 业务需求规格 |
| **核心理念** | 顺应人性的“懒惰”，打造现实版的 AI 数字管家 (Jarvis) |
| **目标用户** | 追求高效生活但厌倦繁琐操作的大学生及年轻职场人 |

## **1\. 产品愿景与核心概念**

UniLife 不仅仅是一个日历工具，它是一个**具备自主决策能力的 AI 生活管家**。

传统的日历是“死的”，需要用户手动填空；UniLife 是“活的”，它理解用户的精力状态、任务的急迫程度以及社交关系。它的核心目标是：**消除“认知摩擦”，让用户从繁琐的日程安排和社交沟通中解放出来，专注于生活本身。**

### **核心差异化体验**

1. **零摩擦录入**：你只需动嘴，剩下的交给 AI。  
2. **顺应人性的排程**：不在你疲惫时安排烧脑任务，顺应你的生物钟。  
3. **A2A (Agent-to-Agent) 社交**：你的 AI 助理去和朋友的 AI 助理谈时间，你只需要做最后的确认。

## **2\. 功能模块详述**

### **2.1 智能交互中心 (Jarvis Interface)**

这是用户与 UniLife 交互的核心入口，位于应用底部的中央悬浮按钮。它模拟了钢铁侠的“Jarvis”体验。

* **全模态输入 (Omni-Input)**：  
  * **语音指令**：用户可以说“帮我约老王下周打球”或“这周五我都空吗？”。  
  * **模糊语义理解**：支持非结构化语言，如“我想找个时间把商业计划书改完”，AI 会自动预估时长并寻找最佳时段。  
  * **截图解析**：用户可以上传教务系统的课表截图、社团活动的宣传海报，AI 自动提取时间地点并生成日程。  
* **突发状况处理 (Emergency Mode)**：  
  * **场景描述**：当用户输入“我生病了”、“发烧要去医院”等指令。  
  * **自动执行**：  
    1. AI 立即启动紧急模式。  
    2. 自动取消或推迟接下来的非紧急个人任务（如背单词）。  
    3. 自动识别涉及他人的行程（如会议、约饭），并触发 **A2A 协议**，自动联系对方的 Agent 进行解释和改期协商。  
    4. 向用户反馈：“已为您取消下午的安排并通知了相关人员，请安心休息。”  
* **主动汇报 (Proactive Briefing)**：  
  * 每天早晨，Jarvis 会主动推送一段语音或文字简报：“早安，今天天气很好。你下午 3 点的状态通常不错，建议把最难的课程设计放在那时处理。”

### **2.2 动态日程管理 (Adaptive Schedule)**

UniLife 的日程不是一张静态的时间表，而是一条流动的河流，根据用户的状态实时调整。

* **E-U 智能排程 (Energy-Urgency Scheduling)**：  
  * **原理**：系统不只看时间空档，更看“精力匹配度”。  
  * **功能表现**：  
    * **深度工作 (Deep Work)**：将高难度的脑力任务安排在用户精力最旺盛的时段（通常标示为 High Energy）。  
    * **自动缓冲 (Smart Buffers)**：在两个任务之间自动插入交通时间和休息缓冲，避免连续作战导致的疲劳。  
    * **疲劳监测**：如果用户连续工作过久，系统会自动建议插入“休息”或“低认知负荷任务”。  
* **双视图展示**：  
  * **流式视图 (Stream View)**：以非线性的方式展示当天的“任务流”，强调先后顺序而非具体时刻，减少时间焦虑。  
  * **月历视图 (Calendar View)**：  
    * 传统的月度网格展示，方便查看宏观安排。  
    * **当日摘要 (Daily Highlights)**：点击月历上的某一天，下方会自动浮现当天的“Top 3 核心事件”卡片，让用户一眼看到当天的重点（如“路演”、“考试”），而不是被琐碎的日程淹没。

### **2.3 A2A 社交协同 (Social & A2A Protocol)**

这是 UniLife 最具颠覆性的功能。它将社交中的“行政工作”外包给了 AI。

* **Agent 代理谈判**：  
  * **场景**：用户想约朋友吃饭，但不知道对方什么时候有空。  
  * **流程**：用户的 Agent 会向朋友的 Agent 发送加密请求。两个 Agent 在后台比对双方的空闲时间、精力状态和偏好地点，最终得出一个或几个“最佳方案”。  
  * **结果**：用户只需在收到的“协商结果卡片”上点击“确认”或“拒绝”。  
* **Agent-Aware 通讯录**：  
  * 在通讯录或消息列表中，用户不再只看到“在线/离线”，而是看到好友的**真实生活状态**（由 Agent 判定，保护隐私的前提下展示）：  
    * 🟣 **深度工作中**：表示对方正在专注，Agent 会自动拦截非紧急消息。  
    * 🔴 **会议中/忙碌**：表示不便打扰。  
    * 🟢 **在线/空闲**：表示适合聊天或发起邀约。  
    * 🟠 **A2A 协商中**：表示 Agent 正在后台处理事务。  
* **社交防火墙 (Social Firewall)**：  
  * **功能**：当收到他人的邀约时，Agent 会根据用户当前的负荷（如本周社交已满、精力透支）进行预判。  
  * **决策权**：AI 不会擅自做主拒绝，而是生成一条**建议**：“检测到你本周社交负荷已达 90%，建议拒绝此聚餐以保留精力复盘。”并提供“强制接受”和“采纳建议（委婉拒绝）”的按钮供用户选择。

### **2.4 个人生活控制台 (Life Console)**

整合了原有的仪表盘和资产页面，为用户提供关于自己生活的全景数据。

* **实时状态看板**：显示当前的精力值（如 High/Med/Low）和下一个建议行动。  
* **生活资产统计**：  
  * **时间储蓄罐**：直观展示 AI 帮你节省了多少小时的“行政与沟通时间”。  
  * **专注力账户**：展示你在高效时段完成核心任务的比例（深度聚焦度）。  
* **生活节奏热力图**：  
  * 以周为单位，展示用户的精力分布。  
  * 清晰标注出哪些天是“高效冲刺日”，哪些天是“恢复休息日”，帮助用户建立张弛有度的生活节奏，避免职业倦怠。

## **3\. 用户故事 (User Stories)**

为了更好地理解功能，以下是几个典型的使用场景：

### **场景 A：突发病假**

**用户行为**：周二早上醒来，Alex 感觉发烧无力，对着手机说：“Jarvis，我发烧了，去不了学校，帮我把今天的事情推掉。”

**系统反应**：

1. Jarvis 识别意图，启动“病假模式”。  
2. 自动取消个人的“晨读”计划。  
3. 检测到下午 2 点有“小组讨论”，Jarvis 自动向组员的 Agent 发送消息：“Alex 突发身体不适，建议会议推迟到周四下午 3 点，各位时间是否方便？”  
4. 几秒后，Jarvis 回复 Alex：“已为您安排好。小组讨论正在协调改期至周四。已为您备注去校医院。请好好休息。”

### **场景 B：无摩擦社交**

**用户行为**：Alex 想约 Sarah 聊项目，但他不想来回发微信确认时间。他对 Jarvis 说：“帮我约 Sarah 下周聊聊 BP，大概一小时。”

**系统反应**：

1. Jarvis 后台联系 Sarah 的 Agent。  
2. 发现 Sarah 周一全天满课，周二下午空闲但精力值预测较低（刚考完试）。  
3. 最终锁定周三上午 10 点，这是双方精力都充沛的时段。  
4. Alex 收到卡片：“已与 Sarah 的 Agent 确认，下周三 10:00 会议，地点建议在图书馆咖啡厅。”

### **场景 C：精力保护**

**用户行为**：Alex 收到一个“校友聚餐”的群邀请。

**系统反应**：

1. Jarvis 分析 Alex 本周日程，发现他已经连续熬夜两天，且明天有重要路演。  
2. Jarvis 弹出提示：“建议拒绝。检测到你处于‘精力透支’边缘，且明天路演优先级为 High。是否需要我帮你委婉推辞？”  
3. Alex 点击“采纳建议”，Jarvis 自动生成礼貌的回复文案发送到群里。

## **4\. 未来展望 (Future Roadmap)**

* **生活服务接入**：当 Agent 安排了“去医院”的日程后，能自动对接打车软件叫车；安排了“约会”后，能自动预订餐厅。  
* **个性化人格**：用户可以定制 Jarvis 的性格（如毒舌激励型、温柔保姆型、严厉导师型），使其不仅仅是工具，更是情感伴侣。  
* **IoT 联动**：与智能家居联动，当检测到用户进入“深度睡眠”日程时，自动调暗灯光、开启勿扰模式。
