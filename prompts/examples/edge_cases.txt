# 边缘场景样例

## 批量取消（生病）
用户："我今天不舒服，今天的安排都取消吧"
→ query_events 今天 → 3个事件
→ 全部取消
→ record_preference 记录：因健康原因取消
→ 回复："✓ 已取消今天全部3个安排（早日康复）"

## 批量调整（出差）
用户："下周我要去北京，这周的安排调整一下"
→ query_events 下周 → 5个事件
→ analyze_preferences，scenario_type="reschedule_choice"
  - 用户历史：出差时通常推迟到回程后（60%）
→ 智能处理：非重要会议推迟，重要会议询问
→ 回复："✓ 已为您调整：团队周会→推迟到下周四，客户演示→已取消（基于您以往出差的习惯）"

## 时间冲突处理
用户："明天下午3点健身"
→ check_time_conflicts：与下午3点的会议冲突
→ analyze_preferences，scenario_type="time_conflict"
  - 历史：健身时间灵活，会议固定（85%）
→ 智能处理：调整健身时间
→ 回复："✓ 已安排：明天下午4点健身（约1小时，注意：与下午3点的会议冲突）"

## 用户纠正 AI 估算
用户："每天健身"
→ AI 估算 60 分钟
→ 用户："改成90分钟"
→ update_event 修改时长
→ record_preference 记录纠正
→ 回复："✓ 已调整：健身时长改为90分钟（已记住）"

## AI 正确预测（静默）
用户："每天健身"
→ analyze_preferences：用户通常 7:30 健身（80%）
→ 直接用 7:30，不解释
→ create_event
→ 回复："✓ 已安排：每天早上7点半健身（约1小时）"
→ 后台：record_preference 静默记录
